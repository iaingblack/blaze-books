Create `BlazeBooks/Models/SchemaV2.swift` with a new `SchemaV2` enum conforming to `VersionedSchema`:
- `static var versionIdentifier = Schema.Version(2, 0, 0)`
- `static var models` array includes Book, Chapter, ReadingPosition, Shelf (all as SchemaV2 types)
- Copy all three existing model classes (Book, Chapter, ReadingPosition) from SchemaV1 into SchemaV2 with ONE addition to Book: add `@Relationship(deleteRule: .nullify, inverse: \Shelf.books) var shelves: [Shelf]? = []`
- Add new `Shelf` model class:
  - `var id: UUID = UUID()`
  - `var name: String = ""`
  - `var createdDate: Date = Date()`
  - `var sortOrder: Int = 0`
  - `@Relationship(deleteRule: .nullify) var books: [Book]? = []`
  - Empty `init() {}` and convenience init with name parameter
- CRITICAL: Use `.nullify` delete rule on BOTH sides of the Book-Shelf relationship (per research Pitfall: cascade on many-to-many would delete books when shelf is deleted). Default arrays to `[]` (per research iOS 17.0 alphabetical ordering bug workaround).

Move the `BlazeBooksMigrationPlan` from SchemaV1.swift into SchemaV2.swift (or keep in SchemaV1.swift, either works), and update it:
- `static var schemas: [any VersionedSchema.Type] = [SchemaV1.self, SchemaV2.self]`
- `static var stages: [MigrationStage] = [.lightweight(fromVersion: SchemaV1.self, toVersion: SchemaV2.self)]`

Update `BlazeBooks/Models/Book.swift` typealias to point to SchemaV2: `typealias Book = SchemaV2.Book`

Create `BlazeBooks/Models/Shelf.swift` with: `typealias Shelf = SchemaV2.Shelf`

Update `BlazeBooks/App/BlazeBooksApp.swift` ModelContainer init to use SchemaV2 model types:
- Replace `SchemaV1.Book.self, SchemaV1.Chapter.self, SchemaV1.ReadingPosition.self` with `SchemaV2.Book.self, SchemaV2.Chapter.self, SchemaV2.ReadingPosition.self, SchemaV2.Shelf.self`

Do NOT modify SchemaV1 model definitions -- they must remain frozen for migration to work. Only move or update the MigrationPlan.
