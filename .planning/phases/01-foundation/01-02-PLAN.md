  <action>
Create `EPUBImportService` as an `@Observable` class. This service handles the full import flow from file picker URL to persisted Book record.

**Properties:**
- `isImporting: Bool = false`
- `importError: String?`
- `importSuccess: Bool = false`

**Main method: `importEPUB(from url: URL, modelContext: ModelContext) async`**

Flow:
1. Set `isImporting = true`, clear previous error/success. Defer setting `isImporting = false`.
2. Call `url.startAccessingSecurityScopedResource()`. If fails, set error "Cannot access the selected file." and return.
3. Defer `url.stopAccessingSecurityScopedResource()`.
4. Compute file hash using `FileStorageManager.computeFileHash(at: url)`.
5. **Duplicate detection:** Query ModelContext for existing Book with matching `fileHash`. If found, set error "Already in your library" and return. Do NOT create a second copy. (Per CONTEXT.md locked decision.)
6. Copy file to `FileStorageManager.booksDirectory` using the original filename. If a file with the same name exists (but different hash — i.e., different book), append a UUID suffix to the filename.
7. Hand the local URL to `EPUBParserService` to extract metadata and chapters.
8. Create a `Book` record with extracted metadata (title, author, cover, filePath as relative path from Documents/Books/, fileHash, chapterCount).
9. Create `Chapter` records for each extracted chapter (title, index, wordCount).
10. Create a `ReadingPosition` record initialized to chapter 0, word 0.
11. Insert all records into ModelContext.
12. Set `importSuccess = true`.

**Error handling per CONTEXT.md decisions:**
- If Readium fails completely (corrupted/encrypted/no content): delete the copied file, set error "Couldn't open this book. It may be damaged or DRM-protected.", do NOT add to library.
- If some chapters fail but others succeed: import succeeds. Set `importSuccess = true`. Failed chapters get wordCount 0 and are marked (store a `parseError` string on Chapter model — add this field if not present, or use a sentinel like wordCount = -1 to indicate broken chapters).

**Smart metadata fallbacks per CONTEXT.md:**
- Missing title -> use filename (without .epub extension)
- Missing author -> "Unknown Author"
- Missing cover -> nil (the view layer handles placeholder generation — Claude's discretion)

Use `@MainActor` on the class since it drives UI state. Parse/copy work happens in async tasks but state updates are on main actor.

**EPUBParserService.swift:**

Create an `@Observable` class that wraps Readium for EPUB parsing and text extraction.

**Readium setup (per research code examples):**
- Private `httpClient = DefaultHTTPClient()`
- Private lazy `assetRetriever = AssetRetriever(httpClient: httpClient)`
- Private lazy `publicationOpener = PublicationOpener(parser: DefaultPublicationParser(httpClient: httpClient, assetRetriever: assetRetriever, pdfFactory: DefaultPDFDocumentFactory()))`

**Properties:**
- `isParsing: Bool = false`
- `parseProgress: Double = 0.0`

**Method: `openEPUB(at fileURL: URL) async throws -> Publication`**
- Open EPUB using `assetRetriever.retrieve(url:)` then `publicationOpener.open(asset:allowUserInteraction:false)`.
- Handle .failure by throwing the error.

**Struct: `ParsedBook`**
- title: String
- author: String
- coverData: Data?
- chapters: [ParsedChapter]

**Struct: `ParsedChapter`**
- title: String
- index: Int
- text: String (plain text content)
- tokens: [WordToken]
- parseError: Bool (true if text extraction failed for this chapter)

**Method: `parseEPUB(at fileURL: URL) async throws -> ParsedBook`**
1. Call `openEPUB(at:)` to get Publication.
2. Extract metadata: `publication.metadata.title`, `publication.metadata.authors.first?.name`, cover data from `publication.cover` (which returns UIImage? — convert to JPEG Data).
3. Extract chapters using `publication.tableOfContents`. If TOC is empty, fall back to `publication.readingOrder` with auto-generated titles ("Section 1", "Section 2", ...). Per research Open Question 2.
4. For each chapter link, get content using `publication.content(from: locator)`. Extract text from `TextualContentElement` elements. Join with newlines.
5. If a chapter yields empty text, try fallback: access raw resource and do basic HTML tag stripping as a last resort. If still empty, mark chapter as `parseError: true` and set text to "This chapter could not be displayed". Per CONTEXT.md locked decision on partially broken books.
6. Tokenize each chapter's text using `WordTokenizer`.
7. Update `parseProgress` after each chapter (index / total).
8. Return `ParsedBook`.

**Handle the "experimental" Content API (research Open Question 1):** Wrap content extraction in do/catch. If it throws, fall back to raw resource access. This makes the parser resilient.

**WordTokenizer.swift:**

Create a struct (not @Observable — it's stateless and synchronous).

**Struct: `WordToken`**
- index: Int
- text: String
- range: Range<String.Index>
- isSentenceEnd: Bool

**Method: `func tokenize(_ text: String) -> [WordToken]`**
Follow the research code example exactly:
1. Create `NLTokenizer(unit: .sentence)`, set string and language (.english), enumerate to collect sentence end indices.
2. Create `NLTokenizer(unit: .word)`, set string and language (.english), enumerate tokens.
3. For each word token, check if any sentence end falls within or at the end of the word's range.
4. Build and return `[WordToken]` array.

Pin to `.english` per research recommendation (Pitfall 5) for deterministic tokenization. This can be made configurable in future phases.

**Method: `func verificationSnippet(tokens: [WordToken], at index: Int) -> String`**
Return the 3-word window around the given index (or fewer at boundaries). Used by ReadingPosition for resilient position restore per research Pitfall 5.
