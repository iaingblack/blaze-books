**ReadingPositionService.swift:**
Create an `@Observable` class for tracking and persisting reading position.

**Properties:**
- `currentChapterIndex: Int = 0`
- `currentWordIndex: Int = 0`
- `progress: Double = 0.0` (0.0 to 1.0, chapter progress)
- `overallProgress: Double = 0.0` (0.0 to 1.0, book progress)

**Methods:**
- `func loadPosition(for book: Book)` — Read the book's `ReadingPosition` from SwiftData. Set `currentChapterIndex` and `currentWordIndex`. If no position exists, create one at (0, 0).
- `func savePosition(book: Book, chapterIndex: Int, wordIndex: Int, tokens: [WordToken], modelContext: ModelContext)` — Update the book's `ReadingPosition` in SwiftData. Also save `verificationSnippet` using `WordTokenizer.verificationSnippet(tokens:at:)`. Update `lastReadDate`. Debounce saves: only actually write to SwiftData if more than 2 seconds since last save (use a timer or timestamp check) to avoid excessive writes during scrolling.
- `func updateProgress(scrollOffset: CGFloat, totalHeight: CGFloat, chapter: Chapter, totalChapters: Int)` — Calculate and update `progress` and `overallProgress`.

**ReadingView.swift:**
A functional reading scaffold per CONTEXT.md: plain scrollable text with basic styling.

**Props:** Takes a `Book` parameter.

**On appear:**
1. Load reading position via `ReadingPositionService.loadPosition(for: book)`.
2. Load chapter text from the `Chapter.text` field in SwiftData. The chapter text was stored at import time (Plan 01-02), so no EPUB re-parsing is needed. Load the current chapter's text into a local `@State` variable. When navigating chapters, update this variable with the new chapter's `text` property.

**Layout:**
- `ScrollViewReader` wrapping a `ScrollView` containing the chapter text.
- Chapter title as a styled header (larger font, bold) at the top.
- Body text with readable typography: system font, ~17pt, comfortable line spacing (1.4-1.6x). Claude's discretion on exact styling.
- Text is broken into paragraphs (split on double newlines from extracted text).
- Each paragraph has an ID for ScrollViewReader scrolling.

**Position tracking:**
- Use `GeometryReader` or scroll position tracking (iOS 17's `scrollPosition` modifier or a `PreferenceKey` approach) to detect the visible portion of the text.
- Map visible offset to approximate word index based on text layout. This is approximate for Phase 1 — exact word-level tracking comes in later phases with RSVP. For now, save position as a chapter-level scroll offset percentage that maps to the nearest paragraph.
- Auto-save position as user scrolls via `ReadingPositionService.savePosition(...)`.

**Position restore:**
- On appear, scroll to the saved position using `ScrollViewReader.scrollTo()`.
- Use the `verificationSnippet` to verify the position is approximately correct (log a warning if mismatch — full search-nearby logic can be enhanced later).

**Progress bar:**
- Thin progress bar at the top or bottom of the reading view (Claude's discretion on positioning) showing `ReadingPositionService.progress`. Use a simple `ProgressView` or custom bar with `.frame(height: 3)`.

**Chapter navigation (basic):**
- Provide previous/next chapter buttons (or swipe) so the user can move between chapters. Update the loaded text and save new position when changing chapters.
- Chapter titles appear as styled headers per CONTEXT.md.

**Broken chapters:**
- If a chapter has `wordCount == 0` or is marked as broken (from Plan 02), display "This chapter could not be displayed" as placeholder text per CONTEXT.md locked decision.
