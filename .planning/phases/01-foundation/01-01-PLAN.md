  <action>
Create SwiftData models following the VersionedSchema pattern from research (Pattern 3). All models MUST be CloudKit-compatible:
- All properties have default values or are optional
- All relationships are optional
- NO @Attribute(.unique) — CloudKit does not support it

**SchemaV1.swift:**
Define `enum SchemaV1: VersionedSchema` with `versionIdentifier = Schema.Version(1, 0, 0)` containing:

- **Book**: id (UUID, default UUID()), title (String, default ""), author (String, default ""), filePath (String, default ""), coverImageData (Data?), importDate (Date, default Date()), chapterCount (Int, default 0), fileHash (String, default "" — for duplicate detection). Relationships: chapters ([Chapter]?, cascade delete, inverse \Chapter.book), readingPosition (ReadingPosition?, cascade delete, inverse \ReadingPosition.book). Add `init()` and a convenience `init(title:author:filePath:...)`.

- **Chapter**: id (UUID), title (String), index (Int), wordCount (Int), book (Book?). Add `init()` and convenience init.

- **ReadingPosition**: id (UUID), chapterIndex (Int, default 0), wordIndex (Int, default 0), lastReadDate (Date, default Date()), verificationSnippet (String, default "" — 3-5 words around position for resilience per research Pitfall 5), book (Book?). Add `init()` and convenience init.

Define `enum BlazeBooksMigrationPlan: SchemaMigrationPlan` with schemas: [SchemaV1.self] and empty stages.

**Book.swift, Chapter.swift, ReadingPosition.swift:**
Create typealiases: `typealias Book = SchemaV1.Book` (same for Chapter, ReadingPosition). This lets the rest of the app use `Book` without knowing about schema versions.

**FileStorageManager.swift:**
Create a utility struct (not @Observable — it's stateless) with:
- `static var booksDirectory: URL` — returns `Documents/Books/`, creating directory if needed
- `static func localURL(for fileName: String) -> URL` — returns full path within booksDirectory
- `static func fileExists(_ fileName: String) -> Bool`
- `static func deleteFile(_ fileName: String) throws`
- `static func computeFileHash(at url: URL) throws -> String` — SHA256 hash for duplicate detection (use CryptoKit)

**BlazeBooksApp.swift (update):**
Wire up the ModelContainer with SchemaV1 models and BlazeBooksMigrationPlan. Use `ModelConfiguration(cloudKitDatabase: .private("iCloud.com.blazebooks.app"))`. Inject via `.modelContainer()` modifier. Inject any @Observable services via `.environment()`.

IMPORTANT: For Phase 1, use `.modelContainer(for: ...)` with local-only config for development. The CloudKit database identifier is set up but actual CloudKit entitlements are Phase 7. Use `ModelConfiguration("BlazeBooks")` without cloudKitDatabase for now — add the CloudKit param in Phase 7. This avoids needing CloudKit entitlements to build.
