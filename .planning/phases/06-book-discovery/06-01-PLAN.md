  <name>Task 2: Create GutendexService API client with pagination and caching</name>
  <files>
    BlazeBooks/Services/GutendexService.swift

  <action>
    Create GutendexService as @MainActor @Observable final class (matching EPUBImportService pattern).

    **Observable state:**
    - `var isLoading = false`
    - `var error: String?`

    **Private state:**
    - In-memory cache: `private var cache: [String: CachedResponse] = [:]` where CachedResponse is a private struct with `response: GutendexResponse` and `timestamp: Date`
    - Cache TTL: 5 minutes (300 seconds)
    - Base URL: `"https://gutendex.com/books"`

    **Public methods:**

    1. `func fetchBooks(topic: String, page: Int = 1) async -> GutendexResponse?`
       - Build cache key from "\(topic)-\(page)"
       - Check cache: if entry exists and age < 300 seconds, return cached response
       - Set isLoading = true, error = nil, defer isLoading = false
       - Build URL with URLComponents: base URL + query items: topic, languages=en, mime_type=application/epub, page
       - Use `URLSession.shared.data(from:)` async/await
       - Decode with JSONDecoder into GutendexResponse
       - Store in cache
       - On error, set self.error = "Could not load books. Check your connection." and return nil

    2. `func fetchNextPage(from nextURL: String) async -> GutendexResponse?`
       - For infinite scroll: takes the `next` URL string directly from a previous GutendexResponse
       - Convert to URL, fetch with URLSession, decode
       - Cache using the URL string as key
       - Same error handling pattern

    **Design notes:**
    - No external dependencies -- URLSession + JSONDecoder only
    - Per research Pitfall 2: 5-min cache reduces API calls well within rate limits
    - Per locked decision: sort by download count is the Gutendex default order, no additional sort parameter needed
    - Per locked decision: English-only via languages=en parameter
    - Per research: mime_type=application/epub filters to books with EPUB format available
  </action>
