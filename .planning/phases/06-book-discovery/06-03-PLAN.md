Purpose: UAT Test 3 reported "It said refreshing genres again and then I think it timed out. Just a blank page." Output: GutendexService that ignores CancellationError cleanly, and GenreBooksView that distinguishes load failure from empty results with retry UI.
</objective>

  <name>Task 1: Make GutendexService CancellationError-aware</name>
  <files>BlazeBooks/Services/GutendexService.swift</files>
  <action>
In both `fetchBooks(topic:page:)` and `fetchNextPage(from:)`, update the catch block to handle CancellationError separately:

1. Before the existing `catch` block, add a `catch is CancellationError` (or check `error is CancellationError` inside the existing catch) that returns nil WITHOUT setting `self.error`. This ensures SwiftUI .task cancellation during navigation transitions does not pollute the error state or prevent the view from retrying on reappear.

2. The existing generic catch should remain for real network errors (URLError, DecodingError, etc.) and continue to set `self.error = "Could not load books. Check your connection."` and return nil.

Pattern for both methods:
```swift
do {
    let (data, _) = try await URLSession.shared.data(from: url)
    let response = try JSONDecoder().decode(GutendexResponse.self, from: data)
    cache[cacheKey] = CachedResponse(response: response, timestamp: Date())
    return response
} catch is CancellationError {
    // SwiftUI .task cancelled during navigation transition -- not a real error
    return nil
} catch {
    self.error = "Could not load books. Check your connection."
    return nil
}
```

Important: The `defer { isLoading = false }` already at the top of each method handles cleanup for all code paths, so no changes needed there. Do NOT move or duplicate isLoading assignment.

<task type="auto">
  <name>Task 2: Add loadFailed state and retry UI to GenreBooksView</name>
  <files>BlazeBooks/Views/Discovery/GenreBooksView.swift</files>
  <action>
Update GenreBooksView to distinguish between three states: initial loading, load failed (with retry), and genuinely empty results.

1. Add a new @State property:
   ```swift
   @State private var loadFailed = false
   ```

2. Update `loadInitialBooks()` to only set `isInitialLoad = false` on success, and set `loadFailed = true` on failure:
   ```swift
   private func loadInitialBooks() async {
       guard books.isEmpty else { return }
       loadFailed = false

       let response = await gutendexService.fetchBooks(topic: genre.topic, page: 1)
       if let response = response {
           books = response.results
           nextPageURL = response.next
           isInitialLoad = false
       } else if !Task.isCancelled {
           // Only treat as failure if the task was not cancelled
           // (cancelled tasks will re-run when SwiftUI re-creates .task)
           loadFailed = true
           isInitialLoad = false
       }
       // If Task.isCancelled, leave isInitialLoad = true so the
       // next .task invocation retries automatically
   }
   ```

3. Update the body to add a failed state branch between the initial loading and the empty state. The view logic should be:
   - `isInitialLoad && books.isEmpty` -> ProgressView (loading spinner) -- existing, keep as-is
   - `loadFailed && books.isEmpty` -> NEW: error message with retry button
   - `books.isEmpty` (neither initial load nor failed) -> "No books found" -- existing empty state
   - else -> LazyVGrid with books -- existing, keep as-is

   The new failed state UI:
   ```swift
   } else if loadFailed && books.isEmpty {
       VStack(spacing: 16) {
           Spacer(minLength: 80)
           Image(systemName: "wifi.exclamationmark")
               .font(.system(size: 40))
               .foregroundStyle(.secondary)
           Text("Could not load books")
               .font(.headline)
               .foregroundStyle(.secondary)
           Text("Check your connection and try again.")
               .font(.subheadline)
               .foregroundStyle(.tertiary)
           Button("Retry") {
               isInitialLoad = true
               loadFailed = false
               Task { await loadInitialBooks() }
           }
           .buttonStyle(.borderedProminent)
           Spacer()
       }
       .frame(maxWidth: .infinity)
   }
   ```

4. Also remove the `guard books.isEmpty else { return }` at the top of loadInitialBooks and replace it with a check that accounts for retry:
   ```swift
   guard books.isEmpty && !loadFailed else {
       if loadFailed { /* allow retry */ } else { return }
   }
   ```
   Actually simpler: just keep `guard books.isEmpty else { return }` since the Retry button resets books would still be empty. The guard is fine as-is because retry only happens when books is empty. Keep the original guard.

Key insight: When Task.isCancelled is true (SwiftUI cancelled the .task during navigation), we leave isInitialLoad = true and do NOT set loadFailed. This means the view stays showing the loading spinner, and when SwiftUI re-invokes .task (which it will after navigation settles), loadInitialBooks runs again automatically because isInitialLoad is still true and books is still empty.
