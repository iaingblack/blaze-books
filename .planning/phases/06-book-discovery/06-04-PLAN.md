**DiscoveryView.swift -- Remove all API preloading:**

1. Remove the `@State private var genreCovers: [String: [URL]]` property entirely
2. Remove the `@State private var isLoadingCovers` property entirely
3. Remove the `.task { await loadGenreCovers() }` modifier entirely
4. Remove the entire `loadGenreCovers()` method entirely
5. Remove the `@Environment(GutendexService.self) private var gutendexService` since DiscoveryView no longer needs it
6. Remove the loading spinner conditional (`if isLoadingCovers && genreCovers.isEmpty`) branch
7. The body should ALWAYS show the `LazyVGrid` with `Genre.all` immediately -- no conditional, no loading state
8. Pass an empty array `[]` for coverURLs to GenreCardView (the fallback gradient will render):
   ```swift
   GenreCardView(genre: genre, coverURLs: [])
   ```
   This means every genre card always shows the fallback gradient background with the SF Symbol icon -- instant, no network dependency.

The resulting DiscoveryView should be roughly 30-40 lines: a ScrollView with a LazyVGrid iterating Genre.all, each wrapped in NavigationLink(value: genre), with the .navigationDestination(for: Genre.self) routing to GenreBooksView. No async code at all.

**GutendexService.swift -- Three performance fixes:**

1. **Fix trailing slash** (line 10): Change `baseURL` from `"https://gutendex.com/books"` to `"https://gutendex.com/books/"` -- this eliminates the 301 redirect round-trip on every request.

2. **Remove mime_type filter** (lines 31-36): In `fetchBooks`, remove the `URLQueryItem(name: "mime_type", value: "application/epub")` from the queryItems array. Keep `topic` and `languages=en` filters. The mime_type filter nearly doubles API response time (46s to 92s per the debug session). Most Gutenberg books have EPUB format anyway, and books without EPUB will simply have `epubURL == nil` (already handled by the optional in GutendexBook).

3. **Add 30-second timeout**: Create a dedicated URLSession configuration with a 30-second timeout instead of using URLSession.shared (which has a 60-second default that allows indefinite waits on a slow API). Add a private lazy property:
   ```swift
   private let session: URLSession = {
       let config = URLSessionConfiguration.default
       config.timeoutIntervalForRequest = 30
       config.timeoutIntervalForResource = 60
       return URLSession(configuration: config)
   }()
   ```
   Replace both `URLSession.shared.data(from: url)` calls (in `fetchBooks` and `fetchNextPage`) with `session.data(from: url)`.
  </action>

<task type="auto">
  <name>Task 2: Ensure GenreBooksView filters out books without EPUB downloads</name>
  <files>
    BlazeBooks/Views/Discovery/GenreBooksView.swift
  </files>
  <action>
Since we removed the `mime_type=application/epub` server-side filter from GutendexService (Task 1), the API may now return some books that don't have an EPUB download available. We need to filter these client-side.

In `GenreBooksView.loadInitialBooks()`, after receiving the response, filter results to only include books with an EPUB URL:
```swift
if let response = response {
    books = response.results.filter { $0.epubURL != nil }
    nextPageURL = response.next
    isInitialLoad = false
}
```

Apply the same filter in `loadNextPage()`:
```swift
if let response = response {
    books.append(contentsOf: response.results.filter { $0.epubURL != nil })
    nextPageURL = response.next
}
```

This preserves the functional equivalence of having the mime_type filter -- users only see downloadable books -- while getting the 2x speed improvement from removing the server-side filter.
  </action>
