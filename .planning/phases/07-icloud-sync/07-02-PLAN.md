    **Create `BlazeBooks/Services/SyncMonitorService.swift`:**

    An `@Observable` service that monitors CloudKit sync state for UI display. Use the `NSPersistentStoreRemoteChange` notification to detect when remote changes arrive.

    ```swift
    import Foundation
    import Observation
    import SwiftData
    import CoreData

    @MainActor
    @Observable
    final class SyncMonitorService {
        /// Whether sync activity is currently happening.
        /// Shows true briefly when remote changes are detected, then resets after a short delay.
        var isSyncing: Bool = false

        private var notificationObserver: NSObjectProtocol?

        /// Start monitoring CloudKit sync events for the given container.
        func startMonitoring(container: ModelContainer) {
            // Observe remote change notifications posted by NSPersistentCloudKitContainer
            notificationObserver = NotificationCenter.default.addObserver(
                forName: .NSPersistentStoreRemoteChange,
                object: nil,
                queue: .main
            ) { [weak self] _ in
                Task { @MainActor in
                    self?.isSyncing = true
                    // Reset after a brief visual indicator period (1.5 seconds)
                    try? await Task.sleep(for: .seconds(1.5))
                    self?.isSyncing = false
                }
            }
        }

        deinit {
            if let observer = notificationObserver {
                NotificationCenter.default.removeObserver(observer)
            }
        }
    }
    ```

    Key design choices:
    - `isSyncing` goes true on remote change notification, then auto-resets after 1.5 seconds. This provides a brief visual pulse each time remote data arrives, matching the user decision for "subtle sync indicator."
    - Uses `.NSPersistentStoreRemoteChange` which SwiftData + CloudKit posts automatically (Research Pattern 3). No custom CloudKit code needed.
    - `object: nil` to catch notifications from any persistent store coordinator (SwiftData doesn't expose the coordinator directly, but the notification is still posted).

    Add the SyncMonitorService environment:

    Add a subtle sync indicator in the toolbar. Per user decision: "small cloud icon or spinner in toolbar while syncing, disappears when done." Add this as a ToolbarItem alongside existing items:
    ```swift
    ToolbarItem(placement: .topBarLeading) {
        HStack(spacing: 16) {
            // Existing items: sortMenu, shelf button, discovery link
            sortMenu
            Button { ... } label: { Image(systemName: "folder.badge.plus") }

                Image(systemName: "icloud")
                    .symbolEffect(.pulse, isActive: true)
                    .foregroundStyle(.secondary)

                    .transition(.opacity)
            }
        }
        .animation(.easeInOut(duration: 0.3), value: syncMonitor.isSyncing)
    }
    ```

    The sync indicator uses SF Symbol `icloud` with a `.pulse` symbolEffect for a subtle animated appearance. It sits alongside existing toolbar items. When `isSyncing` is false, it disappears via opacity transition.

    **Update `BlazeBooks/App/ContentView.swift` -- Guard for undownloaded books:**

    In the NavigationStack's `navigationDestination(for: Book.self)` handler, add a check for `book.isDownloaded`:
    - If `book.isDownloaded` is true: show ReadingView as normal
    - If `book.isDownloaded` is false: show a placeholder view with the book title, a cloud icon, and text explaining the book is still downloading from iCloud. Something like:

    ```swift
    .navigationDestination(for: Book.self) { book in
        if book.isDownloaded {
            ReadingView(book: book)
        } else {
            VStack(spacing: 16) {
                Spacer()
                Image(systemName: "icloud.and.arrow.down")
                    .font(.system(size: 50))
                    .foregroundStyle(.secondary)
                Text("Downloading from iCloud")
                    .font(.title3)
                    .fontWeight(.medium)
                Text(""\(book.title)" is syncing from another device. It will be ready to read once the download completes.")
                    .font(.subheadline)
                    .foregroundStyle(.secondary)
                    .multilineTextAlignment(.center)
                    .padding(.horizontal, 40)
                Spacer()
            }
            .navigationTitle(book.title)
            .navigationBarTitleDisplayMode(.inline)
        }
    }
    ```

    This prevents crashes from trying to open a book without EPUB data. The user sees a friendly message. As CloudKit downloads the CKAsset, `book.epubData` will populate and SwiftUI will reactively update (the SwiftData @Query will refresh).

    Per user decision: "Non-blocking: library shows immediately with whatever has synced, books become readable as they download." This is the natural behavior -- SwiftData populates records as CloudKit delivers them, and the UI is reactive via @Query.
