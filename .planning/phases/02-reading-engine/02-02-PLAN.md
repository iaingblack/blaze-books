**ReadingCoordinator.swift** -- @Observable class, the central state machine for reading:

**Observable state (drives SwiftUI views):**
- `currentWord: ORPWord?` -- the word currently displayed in RSVP view
- `isPlaying: Bool` -- whether reading is active (either mode)
- `isTTSEnabled: Bool = false` -- TTS toggle (default OFF per Claude's discretion: silent RSVP is the primary mode, TTS is the enhancement)
- `currentWPM: Int = 250` -- user's requested WPM
- `effectiveWPM: Int = 250` -- actual WPM after speed cap (may differ from currentWPM when voice is capped)
- `isSpeedCapped: Bool = false` -- whether current voice is capping the speed
- `speedCapMessage: String = ""` -- e.g. "Voice capped at 320 WPM"
- `currentWordIndex: Int = 0`
- `totalWordCount: Int = 0`
- `currentChapterIndex: Int = 0`

**@ObservationIgnored private state:**
- `rsvpEngine: RSVPEngine`
- `ttsService: TTSService`
- `speedCapService: SpeedCapService`
- `chapterTexts: [String]` -- all chapter texts for the book (for auto-advance)
- `totalChapters: Int`

**Initialization:**
- `init(speedCapService: SpeedCapService)` -- create RSVPEngine and TTSService internally. Wire callbacks:
  - `rsvpEngine.onChapterComplete = { [weak self] in self?.handleChapterComplete() }`
  - `ttsService.onWordBoundary = { [weak self] index in self?.handleTTSWordBoundary(index) }`
  - `ttsService.onChapterComplete = { [weak self] in self?.handleChapterComplete() }`

**Public methods:**
- `loadBook(chapterTexts: [String], startChapter: Int, startWord: Int)` -- store chapter texts, set currentChapterIndex, load chapter into both engines: rsvpEngine.loadChapter(text), ttsService.prepareChapter(text). Seek to startWord. Update totalWordCount and totalChapters.
- `play()` -- if isTTSEnabled: start TTS from current word index (ttsService.speak(fromWordIndex:)), TTS word-boundary callbacks will drive currentWord updates. If !isTTSEnabled: start RSVPEngine timer (rsvpEngine.play()), subscribe to its currentWord changes. Set isPlaying = true.
- `pause()` -- pause both engines. rsvpEngine.pause(), ttsService.pause(). Set isPlaying = false. Current word stays frozen (locked decision).
- `resume()` -- back up 4 words from current position (per Claude's discretion, 4 is in the 3-5 range). Calculate backed-up index = max(0, currentWordIndex - 4). If isTTSEnabled: ttsService.stop() then ttsService.speak(fromWordIndex: backedUpIndex). If !isTTSEnabled: rsvpEngine.seekTo(backedUpIndex), rsvpEngine.play(). Set isPlaying = true.
- `stop()` -- stop both engines. rsvpEngine.pause(), ttsService.stop(). Set isPlaying = false.
- `setWPM(_ wpm: Int)` -- store currentWPM = clamp(wpm, 100, 500). Apply speed cap: query speedCapService for effective WPM based on current voice. Update effectiveWPM, isSpeedCapped, speedCapMessage. Apply to engines: rsvpEngine.setWPM(effectiveWPM), ttsService.setRate(speedCapService.wpmToRate(effectiveWPM, for voice)). If slider is capped, snap effectiveWPM to cap value (locked decision: slider shows reality).
- `setTTSEnabled(_ enabled: Bool)` -- update isTTSEnabled. If playing, stop current mode and start the other. If switching to TTS: stop rsvpEngine, start ttsService from currentWordIndex. If switching from TTS: stop ttsService, seek rsvpEngine to currentWordIndex, start rsvpEngine.
- `setVoice(identifier: String)` -- ttsService.setVoice(identifier:). Recalculate speed cap for new voice. Update effectiveWPM, isSpeedCapped, speedCapMessage.

**Private methods:**
- `handleTTSWordBoundary(_ globalIndex: Int)` -- called from TTSService callback. Update currentWordIndex = globalIndex. Set currentWord = rsvpEngine.word(at: globalIndex) to drive the RSVP display with TTS-dictated timing (locked decision: TTS drives everything when active).
- `handleChapterComplete()` -- auto-advance to next chapter with a brief pause. If currentChapterIndex + 1 < totalChapters: set a 1.5-second delay (per Claude's discretion), then load next chapter, reset word index to 0, start playing. If last chapter: stop playback. Use `Task { try await Task.sleep(nanoseconds:) }` for the pause.
- `applySpeedCap()` -- query speedCapService with currentWPM and current voice. Calculate effectiveWPM. If effectiveWPM < currentWPM: set isSpeedCapped = true, speedCapMessage = "Voice capped at \(effectiveWPM) WPM" (locked decision: inline banner). Otherwise: isSpeedCapped = false.

**Key behavioral rules from CONTEXT.md locked decisions:**
1. TTS drives everything when active -- speech dictates word advancement, WPM is approximate
2. Timer drives exact WPM when TTS is off
3. On resume: back up ~4 words for context
4. Chapter auto-advance with brief pause
5. Pause freezes last word on screen
6. Slider snaps to actual capped WPM

**SpeedCapService.swift** -- @Observable class for per-voice speed capping:

**Purpose:** Each Apple voice has a natural maximum WPM. Beyond that, audio becomes garbled. This service detects the cap and clamps WPM gracefully (locked decisions: per-voice cap, slider snaps to reality, inline banner).

**Observable state:**
- `cappedWPM: Int = 500` -- the effective cap for the current voice
- `isCapped: Bool = false`

**@ObservationIgnored private state:**
- `voiceCapCache: [String: Int]` -- maps voice identifier to max WPM. Pre-populated with conservative defaults, updated via calibration.

**Public methods:**
- `maxWPM(forVoice identifier: String) -> Int` -- returns cached max WPM for voice. If not cached, return conservative default (300 WPM for default quality, 350 for enhanced, 400 for premium -- these are starting estimates).
- `effectiveWPM(requested: Int, forVoice identifier: String) -> Int` -- return min(requested, maxWPM(forVoice:)). Simple clamping.
- `wpmToRate(_ wpm: Int, forVoice identifier: String) -> Float` -- convert WPM to AVSpeechUtterance rate (0.0-1.0). Use baseline mapping: rate 0.5 (default) = ~180 WPM. Approximate linear interpolation: rate = 0.5 * (wpm / 180.0), clamped to 0.0-1.0 range. This is an approximation -- exact calibration requires empirical testing (flagged as open question in research). Start with this and refine.
- `rateToWPM(_ rate: Float, forVoice identifier: String) -> Int` -- inverse of wpmToRate. Used for calibration.
- `updateCap(forVoice identifier: String, maxWPM: Int)` -- update cache with empirically measured cap. Called after calibration.

**Silent RSVP (no TTS):** When TTS is off, cap at slider max of 500 WPM (locked decision).

**VoiceManager.swift** -- @Observable class for voice enumeration and management:

**Observable state:**
- `installedVoices: [VoiceInfo]` -- voices currently available on device
- `availableForDownload: [VoiceInfo]` -- enhanced/premium voices not yet installed
- `selectedVoice: VoiceInfo?` -- currently selected voice

**@ObservationIgnored private state:**
- `notificationObserver: Any?` -- for availableVoicesDidChangeNotification

**Public methods:**
- `loadVoices()` -- call AVSpeechSynthesisVoice.speechVoices(). Filter to English voices only (language.hasPrefix("en")) per locked decision (flat list of English voices only for v1). Exclude novelty voices (voiceTraits != .isNoveltyVoice if available, otherwise skip). Separate into installed (all default quality + any enhanced/premium that appear in speechVoices()) and available for download (this is tricky -- Apple doesn't expose a list of downloadable voices; we can only show what's currently installed). Map each to VoiceInfo using VoiceInfo.from(voice:).

  Approach for "Available for Download" section: Since Apple provides no API to list uninstalled voices, the "Available for Download" section should show a guidance card: "More voices are available in Settings > Accessibility > Live Speech > Voices". Tapping opens Settings. When user returns, the notification triggers a reload. Installed voices that are enhanced/premium quality should be marked as such in the installed section.

- `selectVoice(_ voice: VoiceInfo)` -- set selectedVoice. Persist identifier to UserDefaults for next launch.
- `previewVoice(_ voice: VoiceInfo)` -- create a temporary AVSpeechSynthesizer, speak a short sample phrase: "The quick brown fox jumps over the lazy dog." (per Claude's discretion). Use the selected voice's identifier. Stop any currently playing preview first.
- `openVoiceSettings()` -- call UIApplication.shared.open(URL(string: "App-prefs:ACCESSIBILITY")!) to deep-link to Accessibility settings where users can download voices. Fall back to UIApplication.shared.open(URL(string: UIApplication.openSettingsURLString)!) if the accessibility URL scheme fails.
- `startObservingVoiceChanges()` -- register for `AVSpeechSynthesizer.availableVoicesDidChangeNotification` on .main queue. On notification: call loadVoices() to refresh the list.
- `stopObservingVoiceChanges()` -- remove notification observer.
- `loadSelectedVoice()` -- read saved voice identifier from UserDefaults. If found and still available, set selectedVoice. Otherwise default to first installed voice.

**Lifecycle:** Call startObservingVoiceChanges() on init, stopObservingVoiceChanges() in deinit.
