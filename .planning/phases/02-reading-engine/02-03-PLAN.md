**RSVPDisplayView.swift** -- SwiftUI view for ORP-aligned single word display:

Layout follows research code example -- HStack with three Text segments, ORP character anchored at screen center:
- Takes `word: ORPWord?` as input (observed from coordinator)
- HStack(spacing: 0):
  - Text(word.beforeORP) -- right-aligned in fixed-width frame, `.foregroundStyle(.primary)`
  - Text(word.orpCharacter) -- accent color (`.foregroundStyle(.accent)`), `.fontWeight(.bold)`, fixed position at center
  - Text(word.afterORP) -- left-aligned in fixed-width frame, `.foregroundStyle(.primary)`
- Use `.font(.system(size: 36, weight: .medium, design: .monospaced))` for consistent character width
- The left frame width should be calculated to always position the ORP character at the same horizontal center. Use GeometryReader to get view width, set left frame to `viewWidth / 2 - characterWidth / 2`, right frame to `viewWidth / 2 - characterWidth / 2`. Approximate characterWidth from monospaced font metrics.
- When word is nil (idle state): show empty view or a subtle dot/dash indicator
- Add a thin vertical red guide line at the ORP position (center of view) -- subtle visual anchor for the eye
- Wrap in a VStack with generous vertical padding for comfortable reading. Dark background works well for RSVP.
- Smooth transitions: use `.animation(.none)` or `.transaction { $0.animation = nil }` -- RSVP must NOT animate between words, instant swap only.

**VoicePickerView.swift** -- In-reader voice selection sheet (locked decision: accessible from reading view):

Layout:
- NavigationStack with "Voices" title
- Two sections in a List:
  1. **"Installed"** section header -- shows installed VoiceInfo items from voiceManager.installedVoices
     - Each row: voice name, quality badge (enhanced/premium if applicable), tap to select, tap speaker icon to preview
     - Selected voice has a checkmark
     - Tap row -> voiceManager.selectVoice(), also update coordinator's voice
     - Tap speaker icon -> voiceManager.previewVoice() to hear sample phrase (locked decision: fixed sample phrase for comparison)
  2. **"Available for Download"** section -- guidance card (not a voice list, since Apple has no API to enumerate uninstalled voices)
     - Text explaining: "Download enhanced voices for better quality"
     - "Open Voice Settings" button that calls voiceManager.openVoiceSettings()
     - Explanation: "After downloading, return here and new voices will appear automatically"
- Takes `voiceManager: VoiceManager` and closure `onVoiceSelected: (VoiceInfo) -> Void`
- Presented as .sheet from reading view

**SpeedCapBanner.swift** -- Inline non-disruptive banner (locked decision: visible in reading view):

Layout:
- Horizontal bar at top of reading area or below controls
- Shows message like "Voice capped at 320 WPM" with an info icon
- Subtle yellow/orange background with rounded corners
- Only visible when `isSpeedCapped` is true
- Animates in/out with `.transition(.move(edge: .top).combined(with: .opacity))`
- Takes `message: String` and `isVisible: Bool`
